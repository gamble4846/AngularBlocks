Introduction

Welcome to the documentation for Infinite Menu, a powerful Angular library that enables the creation of dynamic and interactive menus with infinite submenus. This library provides developers with an easy-to-use solution for building complex menu structures that can expand indefinitely.

With Infinite Menu, you can take your menu design to the next level by allowing users to navigate through multiple levels of submenus without any limitations. Whether you're building a sophisticated navigation system for a web application or simply want to enhance the user experience on your website, this library has got you covered.

Infinite Menu empowers you to create menus with unlimited levels of submenus. Gone are the days of being restricted by a fixed number of levels, now you can provide your users with a seamless and immersive navigation experience.

Infinite Menu comes with a variety of styling options and configuration settings, allowing you to customize the look and feel of your menus to match your application's design. From colors and fonts to animations and transitions, you have full control over the visual aspects of your menus.

Inputs

1. [(MenuModel)]

@Input() MenuModel: Array<MenuData> = [];

This is a two way binded Model that must contain data for the menu

export interface MenuData {
     Title: string,
     Children: Array<MenuData>,
     MenuLoc?:number,
     IsOpen?:boolean,
     CustomData?:any,
     Id?:string,
     Icon?: string,
     IconType?: IconType,
     OpenSubOnAnyClick?: boolean
}

  1. Title - In this interface, Title is the name of the menu that is displayed in the view
  2. Children - Children is the array of MenuModel that contains all the Children of the current menu
  3. MenuLoc? - MenuLoc (Menu location) is an optional value that is updated by the code and anyvalue added manually will be overwritten, with current location for example, in the menu bar on left side of your screen [Documentation, Food, Drinks ....] are on location 1, Open Documentation and you'll see [Introduction, Getting Started, Installation, Inputs ....] are on location 2, Open Inputs and you'll see [[(MenuModel)] ....] these are on location 3 and so on...
  4. IsOpen? - IsOpen is an optional boolean that specifies if the current menu is open or not meaning if the children of current menu is being displayed or not
  5. Id? - Id is an optional value that is unique for all the menu items, if you dont specify the id manually a random id is generated and added to the menu on initialization
  6. Icon? - Icon is an optional value that can accept, SVG code, Image URL and FluentIcon Name depending upon IconType
  7. IconType? - Icon Type is an optional value that asks for a Enum value from [enum IconType]
  8. OpenSubOnAnyClick? - OpenSubOnAnyClick is an optional boolean that specifies if the children should open even if the user clicks on the title by default this is set to false for all Menu Items

2. Show Open For All

@Input() ShowOpenForAll: boolean = false;

ShowOpenForAll is a boolean value used to specific if the open button (Down Caret on right side of the menu item) should be shown for all menu items or not, by default this is set to false, meaning the open button will be only shown if the menu item contains children

3. Icon Height

@Input() IconHeight: string = "25px";

This is used to set the height of the icon, by default this is set to 25px

4. Icon Width

@Input() IconWidth: string = "25px";

This is used to set the width of the icon, by default this is set to 25px

5. Loading

@Input() Loading: boolean = false;

This is used to Specify if the Menu is loading or not, by default this is set to FALSE

6. Show Only Icon

@Input() ShowOnlyIcon: boolean = false;

This is used to specify whether the title should be shown or not. It is basically used when the menu is collapsed and only icons are needed to be shown. So, if you have something where a small menu (width) is supposed to be shown on a mobile screen, this option is for you.u
